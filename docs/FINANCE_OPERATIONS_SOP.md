# Finance Operations SOP (Standard Operating Procedures)

Purpose: Give the finance team simple, repeatable procedures for running Finance module day-to-day, weekly, and month-end.

Written for beginner-friendly use.

---

## 1) Daily Procedures

### 1.1 Morning Checks (Start of Day)

| Step | Action | Owner |
|------|--------|-------|
| 1 | Login and verify user can access Finance menu | Finance user |
| 2 | Check accounting period is open for current date | Finance admin |
| 3 | Open Day Book for yesterday and confirm postings | Finance user |
| 4 | Review Trial Balance for any unusual balances | Finance lead |

### 1.2 Voucher Processing

| Step | Action | Notes |
|------|--------|-------|
| 1 | Create vouchers for approved transactions | Use balanced lines (debit = credit) |
| 2 | Save as draft for review, or submit if ready | Follow approval rules |
| 3 | Approve submitted vouchers (manager/admin) | Do not approve own vouchers if policy forbids |
| 4 | Post approved vouchers | Only when period is open |

### 1.3 AP/AR and Payments

| Step | Action | Notes |
|------|--------|-------|
| 1 | Create bills for received goods / services | Match PO/GRN where applicable |
| 2 | Process payment runs for approved payables | Approve → Process → Execute |
| 3 | Verify executed runs updated bank and vouchers | Check Payment Advice and linked voucher |

### 1.4 End-of-Day Checks

| Step | Action | Notes |
|------|--------|-------|
| 1 | Ensure all intended vouchers are posted | No orphan drafts for completed work |
| 2 | Run Day Book for today | Archive or export for records if needed |
| 3 | Log any errors or anomalies | Use defect log if issues found |

---

## 2) Weekly Procedures

### 2.1 Bank Reconciliation

| Step | Action | Frequency |
|------|--------|-----------|
| 1 | Download bank statement (CSV or manual) | Per bank, at least weekly |
| 2 | Create or open reconciliation for period | - |
| 3 | Import or add statement lines | - |
| 4 | Run auto-match against payment runs | - |
| 5 | Manual match remaining items | - |
| 6 | Finalize when matched and approved | Manager/admin only |

### 2.2 Outstanding Bills Review

| Step | Action | Notes |
|------|--------|-------|
| 1 | Open AR/AP Aging report | Set correct as-of date |
| 2 | Review overdue buckets (30+, 60+, 90+ days) | Follow up with collections/payments |
| 3 | Export report for records if needed | Use Export CSV |

### 2.3 Cash Forecast

| Step | Action | Notes |
|------|--------|-------|
| 1 | Update cash forecast scenarios if used | Inflow/outflow adjustments |
| 2 | Review Cash Flow Report | Share with finance lead |

---

## 3) Month-End Procedures

### 3.1 Pre-Close Checklist

| Step | Action | Notes |
|------|--------|-------|
| 1 | Complete all reconciliations for the month | All banks finalized |
| 2 | Post all approved vouchers for the month | No pending approvals for month |
| 3 | Run Trial Balance and verify balances | Debit total = Credit total |
| 4 | Run Financial Statements (P&amp;L and Balance Sheet) | Review for reasonableness |
| 5 | Export key reports for month (Day Book, TB, FS) | Archive with dated filename |

### 3.2 Period Close

| Step | Action | Notes |
|------|--------|-------|
| 1 | Get finance lead sign-off for month | - |
| 2 | Close accounting period for the month | Manager/admin only |
| 3 | Verify posting is blocked for closed period | Test with dummy voucher if needed |
| 4 | Create next period if not auto-created | - |

### 3.3 Post-Close

| Step | Action | Notes |
|------|--------|-------|
| 1 | Run Group Summary and Ratio Analysis | For management report pack |
| 2 | Store exported files in designated folder | Backup or share per policy |
| 3 | Update any budget vs actual if budgets are used | Cost center review if enabled |

---

## 4) Exception Handling

| Situation | Action |
|-----------|--------|
| Unbalanced voucher saved by mistake | Edit or delete draft; do not submit |
| Wrong amount posted | Use Reverse action; create correct voucher |
| Period closed too early | Reopen if authorized; reopen only when approved |
| Bank balance mismatch after reconciliation | Re-check statement lines; unmatch and rematch if needed |
| User cannot approve/post | Verify role; escalate to admin for role assignment |

---

## 5) Quick Reference – Report Menu

| Report | Use |
|--------|-----|
| Day Book | Voucher register by date |
| Trial Balance | Ledger debit/credit summary |
| Financial Statements | P&amp;L and Balance Sheet |
| Group Summary | Account group totals |
| Ratio Analysis | Financial health metrics |
| Cash Flow Report | Cash forecast scenarios |
| AR/AP Aging | Outstanding receivables/payables by age |
| Ledger Activity | Account-wise voucher movements |
| Voucher Analytics | Summary, monthly trend, top preparers |

---

## 6) Role Quick Reference

| Role | Typical Access |
|------|----------------|
| Admin | Full access; period close/reopen; user roles |
| Manager | Approve vouchers; finalize reconciliation; close period |
| Finance User | Create vouchers; run reports; payment runs |
| Read-only | Reports only (if role exists) |
