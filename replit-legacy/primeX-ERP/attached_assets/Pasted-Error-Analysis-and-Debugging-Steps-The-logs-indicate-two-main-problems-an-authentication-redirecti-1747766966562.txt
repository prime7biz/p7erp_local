2. React Controlled vs. Uncontrolled Input Warning
Log: Warning: A component is changing an uncontrolled input to be controlled. This is likely caused by the value changing from undefined to a defined value, which should not happen. Decide between using a controlled or uncontrolled input element for the lifetime of the component.

Component Stack: The error originates within an input element, likely wrapped by your custom Input component (/src/components/ui/input.tsx), which is then used within a FormField and Controller from a form library (likely React Hook Form, given FormProvider and Controller in the stack). This is happening inside your QuotationForm (/src/pages/quotations/quotation-form.tsx).

Explanation:

An uncontrolled input is like a traditional HTML form input where the DOM handles the form data. Its React value prop is not set or is undefined.
A controlled input has its value prop managed by React state. An onChange handler updates the state, which in turn updates the input's value.
This warning appears when an input starts as uncontrolled (e.g., its value prop is undefined because the initial data for the form field hasn't loaded yet or is explicitly undefined) and then switches to being controlled (e.g., the data loads, and the value prop receives a defined value like an empty string "" or some actual data). React requires you to pick one strategy for the entire lifecycle of the input.
Possible Causes & Debugging (within QuotationForm):

Initial State: The most common cause is that the initial value for a form field managed by Controller or passed to your Input component is undefined. When data is fetched or state is updated, this value might then become a defined value (e.g., "", null, or an actual string/number).
Default Values: Ensure that your form's default values (e.g., in useForm from React Hook Form if you're using it) provide a defined initial value for every field (e.g., an empty string '' for text inputs, 0 for numbers, null if that's explicitly handled, rather than undefined).
Asynchronous Data: If the form is populated with data fetched asynchronously, make sure that while the data is loading, the input fields have defined initial values.
Conditional Rendering: If the input is conditionally rendered, ensure its value prop is consistent.
Action:

Identify which specific input field in QuotationForm is causing this. The component stack points towards an input used within a FormField.
Inspect the Controller component for that field. Look at its defaultValue prop or the defaultValues provided to useForm.
Ensure the value being passed to the underlying input component (via your custom Input and FormField) is never undefined after its initial render. It should consistently be a string (even an empty one), a number, or null (if you specifically want to treat null as a "defined but empty" state and your input handles it).
For example, if you have value={formData.someField}, make sure formData.someField is initialized to '' or another non-undefined value.
