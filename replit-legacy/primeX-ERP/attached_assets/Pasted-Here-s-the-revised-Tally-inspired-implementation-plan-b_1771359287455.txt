1) Core Principle: One “Party” Master Everywhere (like Tally Party Ledger)
Goal

Sundry Debtors + Sundry Creditors must behave as “Party” (Vendor/Customer/Both) across:

Inventory (PO, GRN, SO, Delivery)

CRM (leads → customers, contacts, history)

Accounts (ledger, vouchers, billing, receivables/payables)

Implementation

Create a single master entity: Party

party_type flags: customer, vendor, both

link to accounting: Party Ledger

link to CRM: contacts, addresses, communication

link to inventory: delivery/ship-to, bill-to, default terms

Tally reference: Party ledgers are grouped under Sundry Debtors/Creditors and used everywhere.

2) Unified Order-to-Cash & Procure-to-Pay (Inventory ↔ CRM ↔ Accounts)
Sales Flow (Customer)

Delivery Challan / Delivery Note (Goods Out)

Sales Invoice (Accounts receivable + revenue + tax)

Receipt / Collection (settlement bill-wise)

Tally supports recording sales against orders and delivery note style goods movement.

Purchase Flow (Vendor)

Purchase Order (PO)

GRN / Receipt Note (Goods In)

Purchase Bill (Accounts payable + expense/stock + tax)

Payment (settlement bill-wise)

3) Delivery Challan + Gate Pass as “Inventory Voucher Workflow” (Key Control Point)
Goal

Delivery Challan & Gate Pass must be the single source of truth for Goods In/Out, with approvals like accounting vouchers.

Design

Create two document types under a shared workflow engine:

DC_OUT (Delivery Challan / Delivery Note) → Goods issue (warehouse to customer/transfer)

GP_OUT (Gate Pass Out) → physical gate clearance control

DC_IN / GRN (Receipt Note) → goods received

GP_IN (Gate Pass In) → physical gate entry control (optional)

Workflow (same pattern as your voucher approval system)

Statuses (example):

DRAFT → SUBMITTED → APPROVED → POSTED → CANCELLED
Rules:

POSTED triggers stock ledger movement and creates accounting impact if configured (or waits for invoice).

Tally reference: Delivery Note voucher exists for delivered goods; workflow/controls are what you’re adding ERP-style.

amount allocated
