Modify the existing “Accounting” module to behave like Tally ERP (voucher-first, ledger-centric, strict double-entry rules, audit trail, and reporting). - Current system is multi-tenant (tenant isolated data). Never leak across tenants.

Non-negotiable Tally-style requirements:
1) Voucher-first workflow:
   - Users create vouchers (Journal, Payment, Receipt, Contra, Sales, Purchase, Credit Note, Debit Note).
   - Voucher numbering per fiscal year + per voucher type + per tenant (configurable prefix).
   - Voucher date must fall within an open fiscal year.

2) Ledger-centric model:
   - Every posting hits Ledgers (which map to Chart of Accounts).
   - Maintain: Ledgers, Groups, Voucher Types, Voucher Entries.
   - Keep “Chart of Accounts” but add “Groups” like Tally:
     Primary groups: Capital Account, Loans (Liability), Current Liabilities, Current Assets, Fixed Assets, Investments, Direct Expenses, Indirect Expenses, Direct Income, Indirect Income, Sales, Purchase, Duties & Taxes, Sundry Debtors, Sundry Creditors, Cash-in-Hand, Bank Accounts.
   - Ledgers belong to a group and can store optional party info (customer/supplier style) for Debtors/Creditors.

3) Double-entry integrity rules:
   - Voucher must have >= 2 lines.
   - Exactly balanced: sum(debit) == sum(credit).
   - Each line must be either debit OR credit (not both).
   - No negative amounts.
   - Enforce at API validation level and again at DB/service layer.
   - Reject invalid payloads with detailed field-level errors.

4) Posting + audit trail:
   - Voucher has status: Draft → Posted (or Approved) → Cancelled.
   - Only Posted vouchers impact Ledger balances.
   - Cancellation should not delete; it should reverse or mark voucher cancelled with preserved lines and audit logs.
   - Track created_by, updated_by, posted_by, timestamps, and “edit history” log entry.

5) Tally-style reports:
   - Day Book (voucher register by date)
   - Ledger Vouchers (ledger drilldown)
   - Trial Balance
   - Profit & Loss
   - Balance Sheet
   - Cash/Bank book
   - Outstanding Receivables/Payables (if ledger is Debtor/Creditor with bill references)
   - Provide APIs + UI pages for each report; allow date range filtering within fiscal year.

6) UX like Tally (web version):
   - Voucher Entry screen is fast keyboard-driven:
     - Choose voucher type
     - Date
     - Narration
     - Lines grid (ledger selection, Dr/Cr toggle, amount)
     - Auto-balance indicator
     - Save Draft / Post
   - Ledger creation screen
   - Group management screen
   - Voucher list screen (filter by type/date/status)
