A) POSTED DOCUMENT IMMUTABILITY + REVERSAL ENGINE
========================================================
1) Identify “posting” entities in this codebase (at minimum):
   - vouchers/journals (accounting postings)
   - inventory movements (issues/receipts/transfers)
   - manufacturing journals / production receipts
   - jobwork issue/receive
   - export invoice (AR)
   - supplier bill (AP)
   NOTE: If the project uses different names, map them accurately and document.

2) Enforce immutability rules:
   - If status == POSTED (or equivalent), block UPDATE/DELETE at API level for:
     - header fields and all lines
   - Allow only:
     - VIEW
     - REVERSE
     - PRINT / EXPORT
   - Add consistent error message: "POSTED documents cannot be edited. Please reverse and repost."

3) Implement a Reversal Engine (generic, reusable):
   - Create server/services/reversalEngine.ts with functions:
     - reverseVoucher(voucherId, tenantId, userId, reason)
     - reverseInventoryTransaction(txnId, tenantId, userId, reason)
     - reverseManufacturingBatch(batchId, tenantId, userId, reason)
     - reverseJobwork(jobworkId, tenantId, userId, reason)
     - reverseInvoice(invoiceId, tenantId, userId, reason)  (create credit note / contra)
     - reverseSupplierBill(billId, tenantId, userId, reason) (debit note / contra)
   - Each reversal must:
     a) Create a new document with type = REVERSAL (or credit/debit note where relevant)
     b) Copy key metadata from original
     c) Generate opposite lines:
        - accounting lines: debit/credit swapped
        - stock moves: qty sign reversed, warehouse reversed if needed
        - manufacturing: WIP/FG movements reversed
     d) Link original↔reversal:
        - original.reversedById
        - reversal.reversalOfId
     e) Lock both documents from edits
     f) Write audit record with reason, user, timestamp, ip (if tracked)

4) Add “reversal-required edit flow”:
   - On UI for posted docs, replace “Edit” with “Reverse & Create New”:
     - Step 1: Reverse (requires reason + permission)
     - Step 2: Clone original into a new DRAFT document user can edit
     - Maintain links: draft.correctingOfId -> originalId (optional)

5) RBAC:
   - Introduce permissions:
     - can_reverse_documents
     - can_period_override (if period lock exists)
   - Only authorized roles can reverse.
   - Record approver if your approval matrix exists; otherwise require “reason” and log.
