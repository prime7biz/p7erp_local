PART-2: TENANT REGISTRATION (CREATES TENANT + TENANT OWNER)
========================================================

GOAL:
Tenant signup creates a new tenant with status=PENDING and creates a TENANT OWNER user for that tenant.
The tenant cannot login until approved by PLATFORM SUPER ADMIN.

Public Route:
POST /api/public/tenant-signup
Input:
- company_name
- tenant_owner_name
- tenant_owner_email (required)
- password
- plan(optional)

Behavior:
1) Generate company_code automatically (NO user editing):
   - sanitize company_name => uppercase letters only
   - take first 3-5 letters (min 3)
   - append 3-4 random digits
   - retry on collision up to 10 times
2) Create tenant row:
   - status = PENDING
   - store generated company_code
3) Create tenant owner user row:
   - tenant_id = newly created tenant.id
   - role = TENANT_OWNER
   - email = tenant_owner_email (required)
   - email_verified = false
   - username:
       - if user didn’t provide username, auto-set to "owner" or "admin"
       - ensure UNIQUE within the tenant (append digits if needed)
   - password_hash = bcrypt hash
4) Return response:
   - message: “Tenant registered successfully and pending approval.”
   - company_code (read-only display)
   - tenant_status: PENDING
   - IMPORTANT: Do NOT issue tenant login token here.
